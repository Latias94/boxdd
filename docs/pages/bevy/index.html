bevy_boxdd

Fixed-step Bevy ECS integration for boxdd, keeping the native Box2D world in a non-send resource while exposing ECS authoring, messages, ray/AABB entity queries, joints, child colliders, filters, and debug draw command collection.

Back to hub · Crate README · Examples · docs.rs

Model

ECS authoring

Spawn RigidBody, Collider, PhysicsMaterial, velocity, impulse, transform sync, and joint descriptor components. Colliders may live on the body entity or child entities. Runtime native ids are inserted as BoxddBody, BoxddShape, and BoxddJoint.

Native ownership

BoxddPhysicsContext owns the boxdd::World as a Bevy non-send resource. Systems move plain ids, values, and entities through messages instead of moving the native world across threads.

Renderer boundary

The crate collects render-agnostic boxdd::DebugDrawCmd values. Applications and examples decide whether to draw them with Gizmos, meshes, sprites, or external tooling.

Workflows

Need Use Start Here
Basic physics scene RigidBody, Collider, PhysicsMaterial, fixed update transform sync. falling_box_2d.rs
Contact and sensor gameplay Enable event flags on PhysicsMaterial and read Bevy messages after physics steps. contact_events_2d.rs, sensor_events_2d.rs
Entity queries Use BoxddPhysicsContext helpers when you want native ray or AABB overlap data mapped back to Bevy entities. ray_query_2d.rs, overlap_query_2d.rs
App-driven kinematics Set TransformSyncMode::BevyToPhysics for kinematic or static bodies controlled by Bevy transforms. kinematic_platform_2d.rs
ECS-authored joints Spawn JointDescriptor::distance or JointDescriptor::revolute entities that reference two Bevy bodies. joint_bridge_2d.rs
Compound bodies Attach multiple child Collider entities to one parent RigidBody when one logical object needs several shapes. child_colliders_2d.rs
Collision filters Set PhysicsMaterial::filter category and mask bits to make gameplay layers collide only with intended layers. collision_filter_2d.rs
Debug draw inspection Collect DebugDrawCmd values from BoxddPhysicsContext and render or log them in app code. debug_draw_collect_2d.rs, debug_draw_gizmos_2d.rs
